# Project Proposal: MeowCare

## One-Line Description
A RAG-powered cat care knowledge assistant that helps cat owners get reliable, source-cited answers to health, nutrition, and behavior questions.

## The Problem
Cat owners frequently face urgent care questions — "My cat is vomiting, is this serious?", "How much should I feed a 6-month kitten?", "Why does my cat keep scratching the furniture?" — and end up scrolling through unreliable forums or generic search results. There's no single, trustworthy tool that provides concise, expert-backed answers with clear source attribution. MeowCare solves this by building a curated knowledge base of high-quality cat care content and using RAG (Retrieval-Augmented Generation) to deliver accurate, contextualized answers with citations.

This matters to me personally — I'm a cat owner and pet content creator with 20k+ followers. I've seen firsthand how much misinformation exists in the pet care space.

## Target User
Cat owners who want quick, reliable answers to care questions — especially first-time cat parents, people who just adopted a rescue, or owners dealing with a new health or behavior issue.

## Core Features (v1)
1. **Chat interface** — Users type cat care questions in natural language and receive conversational answers.
2. **RAG-powered retrieval** — Each answer is generated by retrieving relevant passages from a curated cat care knowledge base, then synthesizing them via LLM.
3. **Source citations** — Every answer includes references to the original source documents so users can verify the information.
4. **User authentication & conversation history** — Login/signup via Clerk so each user has their own saved conversations, with the ability to view and continue past chats.

## Tech Stack
- **Frontend:** Next.js — course-recommended, full-stack framework, Claude knows it deeply.
- **Styling:** Tailwind CSS — utility-first, Claude generates it more reliably than custom CSS.
- **Database:** Supabase — stores knowledge base documents (with embeddings), conversation history, and user data. Supabase's pgvector extension enables vector similarity search for RAG retrieval.
- **Auth:** Clerk — simple drop-in authentication, course-recommended.
- **APIs:** Anthropic API (Claude) for answer generation; OpenAI API for text embeddings (or Supabase's built-in embedding support).
- **Deployment:** Vercel — seamless Next.js deployment, course-recommended.
- **MCP Servers:** Supabase MCP (database operations), Playwright MCP (visual testing and verification).

## Stretch Goals
- **Knowledge base admin panel** — A simple UI for the platform administrator to upload, edit, and manage knowledge base documents, enabling continuous content updates without direct database access.
- **Usage analytics dashboard** — Track query volume, popular topics, retrieval quality metrics.
- **Hybrid retrieval + reranking** — Combine vector search with keyword search, then rerank results for better accuracy.
- **Multi-turn context** — The assistant remembers context within a conversation and can handle follow-up questions like "What about for kittens?" after asking about feeding.
- **Feedback mechanism** — Users can thumbs-up/down answers to create a feedback loop for quality improvement.

## Biggest Risk
**Retrieval quality.** If the knowledge base is too small or poorly chunked, the RAG pipeline will return irrelevant passages and the answers will be bad — no matter how good the LLM is. The chunking strategy, embedding quality, and retrieval logic are the core technical challenges. I need to spend time upfront on data preparation (collecting, cleaning, and chunking cat care articles) before the chat experience will feel good.

## Week 5 Goal

The focus this week is infrastructure and data — getting the retrieval pipeline right before building the user-facing chat.
